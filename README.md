# it327-go
A group project assignment covering an unfamilliar language, Go. 

## Report Requirements
- [ ] The context and primary purpose of the language.
    -  When was it created and why?  
- [ ] Which language paradigm(s) does it fit into?
    -  Describe typical characteristics of this paradigm.  
    -  How does it depart from the the typical characteristics?  
- [ ] Provide a grammar for the language in EBNF or comparable form.
    -  This may be a partial grammar in some cases, but must cover all commonly used features of the language.
    -  Consult with me to determine what should be included in the grammar for your language.
    -  Also, provide the rules for valid identifiers and information about operator precedence (this information can be in an alternate form). Do not create your grammar from scratch.
    -  Your job is to find an appropriate grammar and copy it in the report (with appropriate citation).
- [ ] Provide 3-4 parse trees for somewhat complex program statements/blocks.
    -  For example, if the language was Java, an appropriate choice for a parse tree might be a linear nested if else with 3 branches or a for loop with a single statement in the body of the loop or a very
       simple function or class. Remember to make sure that the parse trees are based on the grammar you have provided.
    -  You will need to provide 1 parse tree per member of your group. The parse trees must be provided in a legible and may not be handwritten.
    -  You can create a landscape section of your report document for the trees if they are wide and that would be helpful.
    -  You will submit the “sentences” that you plan to use for your parse trees for feedback by the Parse Tree Sentence due date. The specific code snippets used do not need to come from your sample programs.
- [ ] Describe the design choices reflected in the language mechanisms and explain how each of these choices relates to the purpose of the language. At a minimum, you should include information about:
    - Typing, type checking, and available types
    - Subroutines, including any overloading or polymorphism
    - Parameter passing mechanisms
    - Scope
    - Memory management
    - Records/structs/tuples/objects
    - If OO, encapsulation and information hiding
    - Be sure to reference your sample programs in this section, including numerous codesnippets highlighting the concepts you are discussing.
- [ ] A discussion of any interesting or unique characteristics of this language not covered above.
- [ ] A thorough evaluation of the language design choices, taking into account the purpose and history of the language. This is a subjective evaluation but must be evidence-based.
    - This is not about whether you like the language. It is about how the language design meets the goals of the language
